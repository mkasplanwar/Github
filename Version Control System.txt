Secara Sederhana Git itu merupakan VCS
VCS -> Version Control System

    Dari Wikipedia
    -> Version Control System, disebut juga Revision control system 
    atau source code management adalah sistem yang mengelola perubahan Dari
    sebuah dokumen, program komputer, website dan kumpulan informasi lainnya
    
    1. Sebuah Sistem yang menyimpan "rekaman / snapshot" perubahan
    Pada source code 
    2. memungkinkan bekerja berkolaborasi dengan lebih baik 
    3. Mengetahui Siapa yang melakukan dan kapan sebuah perubahan terjadi
    4. memungkinkan kita untuk kembali ke keadaan sebelum 
    perubahan (checkout)

Kenapa Kita Harus memakai VCS ini?
-> Dengan Menggunakan VCS ini kita bisa mengelola dokumen kita 
tidak secara Manual
-> Menghindari Kesulitan dalam Membuat versi dalam file kita

VCS ini akan sangat berguna jika kita mengerjakan dokumen secara team 
VCS -> Membatu Kolaborasi lebih mudah

==> Kenapa Pakai Version Control System <==
    1. Kita bisa melacak versi atau history dari perubahan
    pda software kita
    2. Kolaborasi
    3. Sharing

==> Apa Beda nya antara Dropbox & Google Drive 
    Karena tidak se fleksibel jika mengelola 
    Source code

==> VCS ==> Git, Subversion (SVN), Mercurial, CVS (Concurrent Version System)
